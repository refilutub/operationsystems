4.*Як буде відбуватись перенаправлення потоків виведення в bash для наступних дій з командами (позначено як cmd) та файлами (позначено як file):


| **Команда**               | **Що виконує команда?**                                                                                                                                         |
|---------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `cmd 1> file`             | Перенаправляє стандартний потік виведення (stdout) команди `cmd` у файл `file`, заміщуючи вміст файлу.                                                         |
| `cmd > file`              | Те саме, що й попередня команда: перенаправляє стандартний потік виведення (stdout) в файл `file`, заміщуючи вміст файлу.                                         |
| `cmd 2> file`             | Перенаправляє потік помилок (stderr) команди `cmd` у файл `file`, заміщаючи вміст файлу.                                                                      |
| `cmd >> file`             | Перенаправляє стандартний потік виведення (stdout) команди `cmd` у файл `file`, додаючи до існуючого вмісту файлу.                                              |
| `cmd &> file`             | Перенаправляє як стандартний потік виведення (stdout), так і потік помилок (stderr) команди `cmd` у файл `file`, заміщаючи вміст файлу.                          |
| `cmd > file 2>&1`         | Перенаправляє стандартний потік виведення (stdout) команди `cmd` в файл `file` і одночасно перенаправляє потік помилок (stderr) в той самий потік (stdout).         |
| `cmd >> file 2>&1`        | Перенаправляє стандартний потік виведення (stdout) команди `cmd` в файл `file`, додаючи до існуючого вмісту файлу, і одночасно перенаправляє потік помилок (stderr) в той самий потік (stdout). |
| `cmd 2>&1 > /dev/null`    | Перенаправляє потік помилок (stderr) в стандартний потік виведення (stdout), а потім перенаправляє стандартний потік виведення в `/dev/null`, фактично ігноруючи його. |
| `cmd 2> /dev/null`        | Перенаправляє потік помилок (stderr) команди `cmd` в `/dev/null`, ігноруючи помилки.                                                                           |
| `cmd1 | cmd2`             | Виводить результат виконання команди `cmd1` через пайп в команду `cmd2`. Результат роботи `cmd1` стає вхідними даними для `cmd2`.                               |
| `cmd1 2>&1 | cmd2`        | Перенаправляє потік помилок (stderr) команди `cmd1` у стандартний потік виведення (stdout), після чого результат (stdout + stderr) передається через пайп в `cmd2`. |

5. **Розгляньте наведені нижче приклади та поясніть, що виконують дані команди та який тип перенаправлення потоків вони використовують:


| **Команда**                                           | **Що виконує команда?**                                                                                                                                                                                                                     | **Який потік перенаправлення?**                               |
|-------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------|
| `$ echo "It is a new story." > story`                 | Виводить текст "It is a new story." в стандартний потік виведення (stdout) і перенаправляє цей потік у файл `story`, заміщаючи його вміст, якщо файл існує.                                                                                  | **stdout -> file**                                            |
| `$ date > date.txt`                                   | Виводить поточну дату та час в стандартний потік виведення (stdout) і перенаправляє цей потік в файл `date.txt`, заміщаючи вміст файлу.                                                                                                      | **stdout -> file**                                            |
| `$ cat file1 file2 file3 > bigfile`                    | Використовує команду `cat`, щоб зчитати вміст файлів `file1`, `file2` та `file3` і перенаправляє їх в стандартний потік виведення (stdout), який потім перенаправляється в файл `bigfile`, заміщуючи вміст файлу.                             | **stdout -> file**                                            |
| `$ ls -l >> directory`                                | Виконує команду `ls -l`, яка виводить детальний список файлів у поточній директорії, і додає результат в кінець файлу `directory`, не заміщаючи його вміст.                                                                                  | **stdout -> file (додавання)**                                |
| `$ sort < file1_unsorted > file2_sorted`               | Команда `sort` сортує вміст файлу `file1_unsorted`. Стандартний потік вводу (stdin) приймає дані з файлу через `<`. Виведений результат сортування перенаправляється в файл `file2_sorted`, заміщаючи його вміст.                                | **stdin <- file, stdout -> file**                             |
| `$ find -name '*.txt' > file.txt 2> /dev/null`         | Команда `find` шукає всі файли з розширенням `.txt` і виводить результат в стандартний потік виведення (stdout), який перенаправляється в файл `file.txt`. Потік помилок (stderr) перенаправляється в `/dev/null`, ігноруючи помилки.           | **stdout -> file, stderr -> /dev/null**                       |
| `$ cat file1_unsorted | sort > file2_sorted`           | Команда `cat` зчитує вміст `file1_unsorted`, передає його через пайп (`|`) команді `sort`, яка сортує дані і перенаправляє відсортований результат у файл `file2_sorted`, заміщаючи його вміст.                                                | **stdout -> file (через пайп)**                               |
| `$ cat myfile | grep student | wc -l`                  | Команда `cat` виводить вміст файлу `myfile`, передаючи його через пайп в команду `grep`, яка шукає рядки, що містять слово "student", і передає результат через пайп в команду `wc -l`, яка підраховує кількість рядків.                          | **stdout -> stdout (через пайпи)**                            |
